# CONTEXT.md

## Product vision

Build a local, independent **Full Digital Twin Prototype** for a turbine factory production environment.

The system should be able to:

1. Import factory-like BOM, order, routing, process, machine, and work center data from CSV.
2. Generate synthetic workforce and materials.
3. Build order trees from BOM/order data.
4. Generate operation queues from routing.
5. Run a live simulation with adjustable time, pause/resume/fast-forward.
6. Model machines, operators, materials, QC, NCR, rework, scrap, outsourcing, packing and shipment.
7. Run a finite-capacity optimizer without AI.
8. Store all important changes as events.
9. Maintain current-state tables for fast dashboard reads.
10. Support scenario fork/replay/snapshot.
11. Show live operations, Gantt schedule, dashboard KPIs, shortages, bottlenecks, risks and optimizer output.

## Scope type

This is not a small MVP. It is a **Full V1 Prototype**.

## Core design principles

```text
Events preserve history.
Current state tables serve fast UI and simulation.
Snapshots allow replay and scenario forks.
Optimizer proposes schedules.
Simulation validates behavior over time.
User can pause, edit, re-optimize, and inspect impact.
```

## Non-goals in V1

```text
- No real AI/ML training or inference
- No Docker
- No login/authentication
- No direct live connection to factory DB
- No message broker
- No material cost model
- No financial optimizer goals
- No batch/lot/serial material traceability
- No operator fatigue model
- No real external API integration
```

## Current factory system context

The current factory system has a planning snapshot generated by `Query_TimeNeed` into `Query_TimeNeedTable`. That table is useful for understanding the existing planning system, but it is not the core of the new simulator.

For this new simulator, use the real system only for seed data, mainly BOM, orders, routing and machine structure.

## Import boundaries

Import from CSV:

```text
bom.csv
bom_parts.csv
orders.csv
order_parts.csv
routings.csv
routing_operations.csv
processes.csv
process_types.csv
machines.csv
work_centers.csv
machine_processes.csv
```

The CSV column names should stay close to the existing SQL Server naming style, such as `BOMParent_Id`, `BOMChild_Id`, `Order_Id`, `Routine_Id`, `RoutineOperation_Id`, `Process_Id`, `Machine_Id`.

## Generated inside simulator

```text
- 500 synthetic operators / technicians by default
- WorkCenter-based distribution proposed from machine/workcenter data
- Random simple skill levels 1..5
- Configurable multi-skill ratio
- WorkCenter-based shift distribution
- Synthetic initial inventory with presets
- Synthetic suppliers with presets
- QC rules
- NCR workflow rules
- Risk rules
```

## Runtime model

Simulation is hybrid:

```text
- Event-based for core production changes
- Tick/refresh based for UI monitoring
```

Internal time unit is minutes. UI may display minutes, hours, days, or shifts.

## Database model

Use SQL Server locally.

Use both:

```text
Event Store
Current State Tables
```

Event store is for replay/audit/history. Current state tables are for fast simulation and dashboard.
